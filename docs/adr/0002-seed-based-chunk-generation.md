# ADR-0002: Seed-Based Chunk Level Generation

## Status
Accepted

## Context
ด่านของเกมเป็นทางปีนแนวตั้ง ต้องการความหลากหลาย (กันการท่องจำด่าน) แต่ก็ต้องการันตีว่าเล่นได้จริง (ไม่มีช่วงกระโดดไม่ถึง) และตาม ADR-0001 ทุก client ต้องมีด่านตรงกันเป๊ะโดยไม่มี authoritative server แจกข้อมูลด่าน

ทางเลือกที่พิจารณา:
- **Procedural เพียวๆ** — หลากหลายสูงสุด แต่ต้องเขียน validator กันด่านพัง ซึ่งซับซ้อนกว่าการออกแบบมือ
- **Handcrafted เพียวๆ** — คุณภาพการันตี แต่เล่นไม่กี่รอบก็จำด่านได้ กลายเป็นเกมท่องจำ
- **Hybrid chunk** — ออกแบบ chunk มือ (ท่อนละ ~1 จอ) แล้วสุ่มลำดับด้วย seed

## Decision
ใช้ Hybrid chunk: ด่าน = ลำดับ chunk ที่สุ่มจาก seed เดียวซึ่งแชร์ให้ทุก client ตอนเริ่มรอบ ทุกเครื่อง generate ด่านเองแบบ deterministic รวมถึงตำแหน่ง Mystery Box ที่ฝังใน chunk

## Consequences
**ดี:**
- Network payload ตอนเริ่มรอบ = seed ตัวเดียว ไม่ต้อง sync ข้อมูลด่าน
- ทุก chunk การันตีว่าปีนผ่านได้เพราะออกแบบมือ
- เพิ่มความหลากหลาย = เพิ่มจำนวน chunk (content pipeline ชัดเจน ขยายง่าย)
- Chunk ติด tag ความยากได้ เรียงจากง่ายไปยากตามความสูงได้ในอนาคต

**เสีย:**
- ต้องมี chunk editor/format ตั้งแต่แรก (แม้จะเป็นแค่ text/JSON grid)
- รอยต่อระหว่าง chunk ต้องมีกติกา (จุดเข้า-ออกมาตรฐาน) ไม่งั้นต่อกันแล้วกระโดดไม่ถึง
- การสุ่มทุกอย่างในเกม (ลำดับ chunk, ไอเทมใน Mystery Box) ต้องใช้ seeded RNG แยกจาก Math.random เพื่อรักษา determinism
